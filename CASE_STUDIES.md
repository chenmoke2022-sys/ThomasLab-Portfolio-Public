## 案例 A：大规模图压测 | Graph Memory Stress Test

### 问题
当图谱规模增长到很大时，检索是否会"塌缩"（目标项排名快速下降或丢失）？

### 方法
- **Golden Memory**：固定的目标节点
- **压测规模**：逐步增加图谱规模（公开不写具体规模）
- **排名指标**：对所有节点做相似度度量并排序（具体度量方式不公开），记录目标节点排名

### 结论（脱敏）
- **Golden Memory Rank**：始终保持**首位**（未出现 retrieval collapse）
- **内存增长**：随规模近似线性增长（公开不写具体数）
- **端到端耗时**：随规模增长而增长（公开不写具体数）

### 如何复核
技术评审时可屏幕共享完整实验日志、CSV 数据、图表。

## 案例 B：意图直达（门控命中）| Direct Intent Routing

### 问题
能否在不走“深推理/慢路径”的情况下，将模糊自然语言意图安全地映射为机器指令？

### 方法
- **轻量门控（快路径）**：将输入意图映射到“受控指令集合”（具体表示与映射方式不公开）
- **安全门控**：满足安全触发条件才允许执行，否则进入慢路径（确认/推理/人工）（阈值与规则不公开）
- **对照组**：Barrier 模式（本地模型推理/确认路径）

### 结论（脱敏）
- **关键样例**：输入 *"Kill the lights"* → 目标 `CMD_ACTION_X`（示例）
  - 触发结果：**命中预期指令**
  - 端到端延迟：慢路径（模型推理/确认）显著高于快路径（门控命中）（公开不写具体数）
  - 能耗/成本：在同等链路下，快路径显著更省（具体口径与测量方法在私有证据包）

### 如何复核
技术评审时可演示门控工具（公开版）+ 对比图（示意/公开版不含精确刻度）。

## 案例 C：门控延迟分布 | Gate Latency

### 问题
在消费级 CPU 上，门控（输入标准化 → 门控判定 → 路由）的延迟分布是什么？

### 方法
- **测试规模**：多次请求采样，形成分布
- **测试条件**：固定一组代表性负载与门控策略（具体参数不公开）
- **对照**：将门控开销与“重推理/远程调用”等慢路径对比，强调量级差异

### 结论（脱敏）
- **门控开销**：整体为**极低开销**；尾部可控（完整分布在私有证据包）

### 如何复核
技术评审时可展示完整延迟分布图与 trace 日志。
